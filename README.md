# 📝 Evaluating Agent (议论文评改智能体)

Evaluating Agent 是一个基于大语言模型（LLM）的智能体作文自动批改系统。它通过模拟真实阅卷人的“语境与读者建模”，对上传的作文图片进行智能识别、语义切片、逻辑图谱构建，并最终生成多层级的诊断报告（表达、逻辑、交际效果）以及精准预测评分。

## ✨ 核心功能 (Core Features)

1. **OCR文本提取**：从学生手写的作文图片中自动提取原文。
2. **虚拟读者建模**：根据真实题目动态构建虚拟读者预期。
3. **高保真文本切片**：将文章进行段落、句群维度的细粒度切分。
4. **多层级深度审阅**：
    - **层级一 (表达与衔接)**：字词句是否规范可解，段落间是否有逻辑衔接。
    - **层级二 (聚焦与图谱)**：抽取中心论点，识别“孤岛”废话节点及推导层级异常。
    - **层级三 (合作与交际)**：评估作文是否有效更新了读者的信息意义，是否回应了读者的行动期望。
5. **实时状态监看**：前端通过 SSE (Server-Sent Events) 获取后台智能体推理的实时进度与节点流式返回。

## 📂 项目结构 (Project Structure)

```text
Evaluating Agent/
├── app/                  # FastAPI 核心业务代码
│   ├── api/routes.py     # 路由控制接口 (上传与SSE流式输出)
│   ├── models/schemas.py # Pydantic 数据模型定义
│   ├── services/         # 核心智能体工作流 (OCR, LLM 服务, 批改逻辑)
│   └── utils/            # 通用工具组件 (包含基于 Redis 的任务队列)
├── static/               # 前端静态展示页面 (HTML, JS, CSS)
├── tests/                # 测试资产 (默认题干和 Prompt)
├── run.py                # 快速启动入口脚本
├── batch_score.py        # 批量评分自动化测试脚本 (含多线程与断线重连)
├── analyze_results.py    # 评估结果的数据分析脚本 (皮尔逊/斯皮尔曼相关性，误差分布)
└── decode_csv.py         # 用于修复含有 Unicode 转义的 CSV 读取修复脚本
```

## 🚀 快速开始 (Quick Start)

### 1. 环境准备
项目使用 `uv` / `pip` 进行包管理：
```bash
# 激活虚拟环境后安装依赖 (如果未安装依赖)
pip install -r requirements.txt # 或者使用 uv
```

*注意：系统后台采用了异步队列处理，请确保本地已在运行 **Redis** 服务 (默认 `app.utils.task_manager` 可能依赖 Redis)。*

### 2. 启动服务 (Web 模式)
在项目根目录运行 FastAPI 后端服务：
```bash
uv run run.py
```
启动成功后，浏览器访问：[http://127.0.0.1:8000/](http://127.0.0.1:8000/) 即可打开前端上传界面。
